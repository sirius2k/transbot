# FEATURE-001: 영어 ↔ 한국어 양방향 번역

## 개요

- **기능명**: 영어-한국어 양방향 번역
- **상태**: ✅ 완료
- **분류**: 백엔드 + 프론트엔드

## 기능 설명

사용자가 입력한 텍스트를 OpenAI API를 통해 영어에서 한국어로, 또는 한국어에서 영어로 번역하는 핵심 기능

## 작업(Task) 분해

### Task 1.1: 백엔드 - OpenAI API 연동 모듈

- **분류**: 백엔드
- **의존성**: 없음
- **작업 내용**:
  - OpenAI 클라이언트 초기화
  - API 키 관리 (환경변수 및 사용자 입력)
  - API 호출 함수 구현
- **예상 파일**: `app.py` (translate 함수)

### Task 1.2: 백엔드 - 번역 프롬프트 설계

- **분류**: 백엔드
- **의존성**: Task 1.1
- **작업 내용**:
  - 시스템 프롬프트 작성 (번역 전문가 역할 부여)
  - 원본 언어 → 대상 언어 변환 로직
  - 번역 품질 향상을 위한 프롬프트 엔지니어링
- **예상 파일**: `app.py` (translate 함수 내 messages 구성)

### Task 1.3: 프론트엔드 - 입력 UI 구현

- **분류**: 프론트엔드
- **의존성**: 없음
- **작업 내용**:
  - 텍스트 입력 영역 (st.text_area)
  - 번역 방향 선택 (영→한, 한→영)
  - 번역 버튼 구현
- **예상 파일**: `app.py` (Streamlit UI 컴포넌트)

### Task 1.4: 프론트엔드 - 결과 출력 UI 구현

- **분류**: 프론트엔드
- **의존성**: Task 1.1, Task 1.2, Task 1.3
- **작업 내용**:
  - 번역 결과 표시 영역
  - 로딩 상태 표시 (st.spinner)
  - 에러 메시지 표시
- **예상 파일**: `app.py` (결과 표시 섹션)

### Task 1.5: 에러 핸들링

- **분류**: 백엔드 + 프론트엔드
- **의존성**: Task 1.1 ~ Task 1.4
- **작업 내용**:
  - API 키 미입력 시 경고
  - API 호출 실패 시 에러 처리
  - 네트워크 오류 처리
- **예상 파일**: `app.py` (try-except 블록)

## 작업 흐름도

```text
Task 1.1 (API 연동) ──┬──→ Task 1.2 (프롬프트) ──┐
                      │                           │
Task 1.3 (입력 UI) ───┴───────────────────────────┴──→ Task 1.4 (출력 UI) ──→ Task 1.5 (에러 핸들링)
```

## 완료 기준

- [ ] 영어 텍스트 입력 시 한국어로 정확히 번역됨
- [ ] 한국어 텍스트 입력 시 영어로 정확히 번역됨
- [ ] API 오류 발생 시 사용자에게 명확한 에러 메시지 표시
- [ ] 번역 진행 중 로딩 상태 표시
