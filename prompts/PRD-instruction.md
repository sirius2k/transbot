# 개요

이 파일은 PRD를 작성하기 위한 참고 내용을 정리해놓은 파일입니다.

## 문제 정의

번역을 해주는 웹앱과 앱들이 많지만 다음과 같은 문제가 있습니다.

- 번역을 하는 기능 자체는 매우 잘 동작하지만 사용자가 원하는대로 UI를 바꾸거나 기능을 추가할 수 없음.
- 여러가지 업무에 쓰이는 고유한 단어를 모르기 때문에 번역이 매끄럽지 않을 수 있음.
- 사전을 연동하여 번역기능 외에도 사전 기능이 없어서 단어의 뜻만 보고 싶을 경우 보기가 어려움.
- LLM을 통해 번역을 요청하지만 어떤 번역을 요청했는지 히스토리를 알 수 없음.
- 로컬 파일, 구글 드라이브의 파일, Confluence등 외부 저장소에 있는 파일의 번역본을 빠르게 만들 수 없음.

## 타겟 사용자 및 사용 사례

### 이 제품을 사용할 핵심 사용자

번역을 필요로 하고, 앱을 자유롭게 만들고 싶은 개발자

### 이 제품의 목표

번역을 원하는 텍스트나 파일등의 원본을 입력했을 때, 즉시 번역하여 결과를 보여줍니다. 번역이 완료되면 그 내용을 별도의 저장소에 기록하여 그것을 계속 관리하여 편리하게 재조회하거나 다른 툴로 내용을 전송하는등의 방법으로 활용할 수 있도록 합니다.


### 제안 해결책

- Streamlit 기반의 UI를 제공하는 앱으로 번역, 저장, 관리까지 모든 것을 해결할 수 있는 솔루션 제공
- 직관적인 UI를 통해 원하는 번역과 사전 검색을 빠르게 할 수 있도록 함.
- 번역하고 싶은 원본 텍스트를 붙여넣기 하여 즉시 그 번역본을 볼 수 있도록 함.
- 번역된 내용을 빠르게 복사하고 다른 곳에 붙여넣기 할 수 있도록 함.
- 기존의 번역 원본 문서의 최대 제한 글자 수가 있는 것을 확장할 수 있도록 함.
- 단어만을 입력했을 때 사전을 검색하여 그 뜻을 빠르게 볼 수 있도록 함.
- 번역 및 사전 내용들의 히스토리를 관리하고, 그룹핑하고, 조회할 수 있도록 함.
- 번역 및 사전 내용들을 외부 툴(Notion, Asana, Confluence등)에 저장할 수 있도록 함.
- LLM을 사용한 내역을 추적할 수 있는 관찰성(Observability) 지표를 확인할 툴(Langfuse)을 연동.
- 빠르게 개발할 수 있는 유연한 프로젝트 구조로 기능을 쉽게 추가할 수 있도록 함.

## 성공 지표요소

- 개발 후 1개월 내에 다른 유료 번역 툴을 쓰는 비중을 10%이내로 줄임.
- 개발 후 번역과 관련된 업무 자동화 비율을 95% 이상 유지.
